# 设计系统

> anime_ui 前端样式与布局的统一规范，新组件开发必须遵循。

## 1. 设计令牌

### 1.1 颜色（AppColors）

**位置**：`anime_ui/lib/pub/theme/colors.dart`

| 类别 | 令牌 | 说明 |
|------|------|------|
| 主色 | `primary`(#9061F9)、`secondary`、`onPrimary` | 强调色、按钮等 |
| 主色状态 | `primaryHover`、`primaryPressed`、`primarySubtle` | hover/pressed/subtle 三种交互态 |
| 背景与表面（紫调） | `backgroundDarkest`、`background`、`surface`、`surfaceContainer`/`High`/`Highest` | 亮度严格递增：background(12) < surface(21) < container(26) < high(32) < highest(40) |
| 蓝灰表面 | `surfaceMuted`、`surfaceMutedDark`、`surfaceMutedDarker` | 蓝灰调容器，用于弹窗、输入框、卡片等 |
| 文字灰度（蓝灰调） | `onSurface` → `mutedLight` → `muted` → `mutedDark` → `mutedDarker` → `mutedDarkest` | 蓝灰调，与紫蓝背景和谐，亮度严格递减 |
| 语义 | `error`、`warning`、`success`、`info`、`newTag`、`tagAmber` | 状态提示、标签 |
| 边框与分割 | `border`(#2A2A44，卡片/面板)、`divider`(#222238，内部分割) | 两者有明确亮度区分 |
| 输入与交互 | `inputBackground`、`inputFill`、`inputBorder`、`chipBgHover`、`chipBorderHover` | 蓝调化，与整体色调统一 |
| 资产分类 | `categoryCharacter`、`categoryLocation`、`categoryProp` 等 | 卡片、标签 |
| 情绪向量 | `emotionHappy`、`emotionAngry`、`emotionSad` 等 | IndexTTS2 8 维情绪色 |
| 角色徽章 | `roleOwnerBg`/`Fg`、`roleDirectorBg`/`Fg` 等 | 权限角色标识 |

**用法**：`AppColors.primary`、`Theme.of(context).colorScheme.primary`

## 2. flutter_screenutil 尺寸适配

**设计稿基准**：1920×1080（`main.dart` 中 `ScreenUtilInit.designSize`）

**字号不缩放**：`ScreenUtilInit.enableScaleText: () => false`，`.sp` 直接使用设计稿字号，避免 Web 亚像素渲染导致字体模糊。

### 2.1 扩展方法

| 扩展 | 用途 | 示例 |
|------|------|------|
| `.w` | 宽度、水平间距 | `SizedBox(width: 24.w)` |
| `.h` | 高度、垂直间距 | `SizedBox(height: 16.h)` |
| `.r` | 圆角、通用尺寸 | `BorderRadius.circular(8.r)` |
| `.sp` | 字号 | `TextStyle(fontSize: 14.sp)` |

### 2.2 使用约定

1. **尺寸**：`SizedBox`、`EdgeInsets`、`width`/`height` 等使用 `.w`、`.h`、`.r`
2. **圆角**：`BorderRadius.circular(X.r)`，X 为数字或 `RadiusTokens.xxx`
3. **字号**：`AppTextStyles` 已内置 `.sp`，自定义时用 `fontSize: 14.sp`
4. **图标**：`Icon(..., size: 20.r)`
5. **面板宽度**：`Spacing.listPanelMinWidth.w`、`Spacing.reviewLeftWidth.w` 等

### 2.3 导入

```dart
import 'package:flutter_screenutil/flutter_screenutil.dart';
```

## 5. 动效令牌（MotionTokens）

**位置**：`anime_ui/lib/pub/const/motion.dart`

### 5.1 时长

| 令牌 | 值 | 用途 |
|------|-----|------|
| durationFast | 150ms | 微交互（hover、ripple） |
| durationMedium | 250ms | 标准交互（展开/折叠、切换） |
| durationSlow | 350ms | 页面过渡、大面积变化 |
| durationEmphasis | 500ms | 强调动效（入场、引导） |

### 5.2 曲线

| 令牌 | 值 | 用途 |
|------|-----|------|
| curveStandard | easeOutCubic | 展开、进入 |
| curveDecelerate | easeOut | 消失、折叠 |
| curveSpring | easeOutBack | 按钮点击、卡片弹回 |
| curveLinear | linear | 进度条、计时器 |
| curveSmooth | easeInOut | 大面积过渡 |

**用法**：`AnimatedContainer(duration: MotionTokens.durationMedium, curve: MotionTokens.curveStandard, ...)`

## 6. 禁止事项

- 禁止硬编码 `Color(0xFF...)`、`Colors.grey`、`Colors.white` 等
- 禁止硬编码 `BorderRadius.circular(8)`，应使用 `RadiusTokens.xxx.r`
- 禁止硬编码 `SizedBox(width: 24)`，应使用 `Spacing.xxx.w` 或 `24.w`
- 禁止硬编码 `TextStyle(fontSize: 14)`，应使用 `AppTextStyles.xxx` 或 `14.sp`

## 7. Lint 提示

### 6.1 项目已启用的相关规则

项目通过 `package:flutter_lints` 引入 Flutter 推荐规则集，其中与设计令牌相关的规则包括：

| 规则 | 说明 |
|------|------|
| `prefer_const_constructors` | 鼓励对可 const 的构造函数使用 `const`，减少 Widget 重建 |
| `prefer_const_constructors_in_immutables` | 对 `@immutable` 类优先使用 const 构造函数 |
| `prefer_const_declarations` | 声明时优先使用 `const` 而非 `final` |
| `prefer_const_literals_to_create_immutables` | 创建不可变对象时优先使用 const 字面量 |
| `use_full_hex_values_for_flutter_colors` | Color 使用 8 位十六进制（如 `0xFFFFFFFF`） |

### 6.2 需手动避免的硬编码模式（无现成 Lint）

Dart/Flutter 官方 linter 目前**没有**禁止硬编码颜色、尺寸的规则（如 `avoid_hardcoded_colors`），以下模式需开发者自行遵守：

| 禁止模式 | 正确用法 |
|----------|----------|
| `Color(0xFF...)`、`Colors.grey`、`Colors.white` | `AppColors.xxx`、`Theme.of(context).colorScheme.xxx` |
| `BorderRadius.circular(8)` | `BorderRadius.circular(RadiusTokens.xxx.r)` |
| `SizedBox(width: 24)`、`EdgeInsets.all(12)` | `Spacing.xxx.w`、`Spacing.xxx.r` 或 `24.w` |
| `TextStyle(fontSize: 14)` | `AppTextStyles.xxx` 或 `14.sp` |
| `Icon(..., size: 20)` | `Icon(..., size: 20.r)` |

代码审查时请重点检查上述模式。
