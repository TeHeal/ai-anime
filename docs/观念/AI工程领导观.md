# AI 时代的工程领导观

> 来源：纽约证券交易所首席技术官 斯里达尔·马萨姆 (Sridhar Masam)

---

## 核心观念摘要

| 维度 | 核心要点 |
|------|----------|
| **实践应用** | 用 Claude Code 重塑工程流程；Claude Agent SDK 构建 AI 代理，接收 Jira 工单指令并生成可提交代码 |
| **责任转移** | 传统：构建确定性平台，需求→构建→上线即止；AI 时代：因 AI 具概率性，责任延伸至**每日监控**其行为与结果 |
| **新范式「组装」** | 超越「购买与自建」；将多模型、多供应商、平台、数据与内部能力整合为解决方案 |
| **风险观** | 受严格监管行业须从**规避风险**转向**风险校准**；单纯回避 AI 已非具竞争力选择 |

**一句话概括**：领导者的思维方式须根本转变——责任不止于上线，需持续监控；采用「组装」范式整合能力；从风险规避转向风险校准。

---

## 原文摘录

「责任正在转移。传统上，我们习惯于构建确定性平台。编写代码需求，然后进行构建。而现在，由于 AI 具有概率性，责任并非止于项目上线，而是需要每日监控其行为和结果。」

「组装」——将多种模型、多个供应商、平台、数据和内部能力整合到解决方案中。

受严格监管的行业必须「从规避风险转向风险校准」，因为仅仅回避 AI 已不再是一种具有竞争力的选择。
