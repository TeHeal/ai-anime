# AI 驱动漫剧开发 — 分镜脚本格式说明（v4.0 增强版）

## 一、设计目标

- **AI 友好**：字段语义清晰、受控词表可校验，便于大模型与自动化流水线解析。
- **流水线可编排**：每镜通过 **生成任务** 标记「本镜需要生成图像 / 视频 / TTS / 音效 / BGM / 转场」等，便于任务调度与进度追踪。
- **主控编排**：保持主控脚本统一，支持时间轴同步、任务依赖管理，避免资源分离导致的上下文丢失。
- **详细任务配置**：支持音频五大流线（VO、BGM、拟声、动态音效、氛围音效）的详细配置，便于ComfyUI Workflow编排。
- **风格与角色一致**：全局 `全局风格`、`默认反向提示词`，单镜可覆盖；`角色ID` 可关联角色设定，保证画风与音色一致。
- **统一标准**：ComfyUI anime 节点及其它下游格式均以此格式为准，后续由下游适配读取本格式。

## 二、顶层结构（v4.0，中文键名）

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| `版本` | string | 是 | 固定 `"4.0"` |
| `格式标识` | string | 否 | 如 `"storyboard_ai_v4"`，便于工具识别 |
| `项目` | string | 是 | 项目/作品名 |
| `集数` | number | 否 | 集数 |
| `集标题` | string | 否 | 集标题，如「第一集」 |
| `受控词表` | object | 否 | 景别/运镜/转场/生成任务等可选值，见下表 |
| `项目元信息` | object | 否 | 全局风格与制作说明，见下表 |
| `镜头列表` | array | **是** | 镜头列表 |

### 受控词表（可选）

| 子键 | 说明 | 示例值 |
|------|------|--------|
| `景别` | 景别可选值 | `["大特写","特写","近景","中景","中远景","全景","远景","大远景"]` |
| `运镜` | 运镜可选值 | `["固定","推","拉","摇","移","跟","升","降","环绕","其他"]` |
| `转场` | 转场可选值 | `["硬切","淡入淡出","闪白","叠化","划变","淡入黑屏","无"]` |
| `优先级` | 镜头优先级 | `["P0必出","P1核心","P2过渡"]` |
| `生成任务` | 本镜可触发的任务类型 | 见下「生成任务」说明 |

### 生成任务（受控词表与单镜取值）

每镜的 **生成任务** 为字符串数组，标明本镜需要执行哪些生成步骤。推荐取值（建议在受控词表中统一）：

| 取值 | 说明 | 流水线用途 |
|------|------|------------|
| **图像** | 本镜需生成静态图/关键帧 | 图像生成（文生图、图生图等） |
| **视频** | 本镜需生成动态视频/动效 | 视频生成、动效、口型等 |
| **TTS** | 本镜需生成对白/旁白语音 | 文本转语音，用 台词+角色+情绪描述 |
| **音效** | 本镜需生成音效 | 剑鸣、风声、脚步等 SFX |
| **BGM** | 本镜需背景音乐 | 背景音乐片段或延续 |
| **转场** | 本镜需生成转场效果 | 闪白、叠化等转场素材 |

示例：`["图像", "视频", "TTS", "音效"]` 表示本镜需要出图、出视频、出对白、出音效；无对白镜头可为 `["图像", "音效"]`。

### 项目元信息（可选）

| 子键 | 说明 |
|------|------|
| `全局风格` | 全局画面风格，供图像/视频模型默认使用 |
| `默认反向提示词` | 全局反向提示词，单镜未写时使用 |
| `制作适配说明` | 节奏把控、运镜规范、音频准则、AI生成要求 |

## 三、单镜字段（v4.0 增强版）

### 基础字段

| 字段 | 类型 | 必填 | 说明 | 流水线用途 |
|------|------|------|------|------------|
| `镜号` | number | **是** | 连续编号，从 1 开始 | 顺序与下游镜号一致 |
| `时长` | number | 建议 | 秒，建议 2–3 | 成片剪辑、节奏 |
| `时间轴` | object | 建议 | 精确时间轴信息 | **资源同步和合成** |
| `优先级` | string | 否 | P0必出/P1核心/P2过渡 | 排期与重跑优先级 |
| `景别` | string | 建议 | 建议在 受控词表.景别 中 | 图像/视频提示 |
| `运镜方式` | string | 建议 | 建议在 受控词表.运镜 中 | 同上 |
| `角色站位` | string | 否 | 角色位置、动作 | 并入画面描述或 AI 提示 |
| `画面描述` | string | **是** | 场景、道具、氛围 | → 图像/视频生成 |
| `台词` | string | 建议 | 对白/OS/旁白，无则「无」 | → TTS |
| `角色` | string | 有台词时建议 | 说该句台词的角色名 | → TTS 音色映射 |
| `角色ID` | string | 否 | 与角色设定表对应 | 立绘/音色/风格绑定 |
| `情绪描述` | string | 否 | 情绪/表演提示 | → TTS 情感、图像表情 |
| `情绪向量` | array(number) | 否 | 情绪多维向量，供TTS/图像情感模型使用 | → TTS 情感、图像表情（与情绪描述文本互补） |
| `音频设计` | string | 否 | 对白/核心音效/环境音说明 | → 音效设计/混音 |
| `AI提示词` | string | 建议 | 运镜+风格+细节，供图像/视频 | → 图像/视频生成 |
| `反向提示词` | string | 否 | 本镜反向提示词，覆盖全局 | → 图像/视频生成 |
| `转场方式` | string | 否 | 建议在 受控词表.转场 中 | → 剪辑/转场 |
| **`生成任务`** | array | 建议 | 本镜需执行的任务：图像/视频/TTS/音效/BGM/转场 | **流水线调度** |
| `备注` | string | 否 | 其他备注 | — |

### 时间轴字段（新增）

| 子字段 | 类型 | 说明 |
|--------|------|------|
| `开始时间` | number | 秒，从0开始累计，用于资源同步 |
| `结束时间` | number | 秒，开始时间+时长，用于资源对齐 |

**用途**：确保音频、图像、视频资源在时间轴上精确对齐，支持自动计算和资源合成。

### 任务配置字段（新增，核心增强）

任务配置字段直接在镜头对象下，与基础字段同一层级，支持ComfyUI Workflow编排：

#### 音频字段（音频五大流线配置）

`音频` 对象包含五大流线的详细配置：

**1. VO (Voice Over) - 旁白、对白、人物同期声**
- `启用`: 是否启用此流线
- `类型`: "TTS" | "同期声" | "旁白"
- `台词`: 对白文本（继承自顶层`台词`字段）
- `角色ID`: 角色标识（继承自顶层`角色ID`）
- `情绪描述`: 情绪描述（继承自顶层`情绪描述`）
- `情绪向量`: 情绪多维向量（继承自顶层`情绪向量`）
- `音量`: 0-1，默认0.8
- `优先级`: 任务优先级

**2. BGM (Background Music) - 背景音乐**
- `启用`: 是否启用此流线
- `类型`: "生成" | "库匹配" | "延续"（延续上一镜的BGM）
- `提示词`: AI音乐生成提示词（当类型为"生成"时使用，如"激昂的战斗音乐，古风，节奏紧张"）
- `风格`: 音乐风格描述
- `情绪描述`: 音乐情绪
- `强度`: 0-1，默认0.6
- `淡入时间`: 秒，默认0.5
- `淡出时间`: 秒，默认0.5

**3. 拟声 (Foley Sound) - 模拟画面中事物的声音**
- `启用`: 是否启用此流线
- `类型`: "生成" | "库匹配"
- `提示词`: AI音效生成提示词（当类型为"生成"时使用，如"金属剑刃快速划过空气的声音，带有风声"）
- `描述`: 音效描述（如"御剑风声"、"剑鸣"）
- `触发时间`: 秒，相对于镜头开始时间
- `音量`: 0-1，默认0.7
- `优先级`: 任务优先级

**4. 动态音效 (Dynamic Sound Effects) - 转场、人物状态、信息暗示等**
- `启用`: 是否启用此流线
- `类型`: "转场" | "状态变化" | "信息暗示"
- `提示词`: AI音效生成提示词（用于生成转场或状态变化音效，如"转场音效，短促的金属撞击声，带有回音"）
- `描述`: 音效描述
- `触发时间`: 秒，相对于镜头开始时间
- `音量`: 0-1，默认0.6

**5. 氛围音效 (Ambient Sound Effects) - 场景、环境音效**
- `启用`: 是否启用此流线
- `类型`: "生成" | "库匹配"
- `提示词`: AI环境音生成提示词（当类型为"生成"时使用，如"空旷的天空环境音，有气流呼啸声和风声，持续循环"）
- `描述`: 环境音描述（如"气流呼啸声"、"街道嘈杂声"）
- `强度`: 0-1，默认0.4
- `循环`: 是否循环播放，默认true

#### 图像字段（图像生成任务配置）

**AI驱动优化方案**：充分利用AI的一次性生成能力，采用**完整画面+叠加特效**的简化结构。

**基础配置**（默认，90%的场景）：
- `启用`: 是否启用图像生成
- `类型`: "静态图" | "关键帧"
- `提示词`: **完整场景提示词**，包含角色、背景、道具等所有元素，AI一次性生成完整画面
  - 示例："2D漫风，拉镜运镜，青云镇全景，顾白御剑凌空，剑尖朝前，色调偏暗，天空白云翻滚"
  - **优势**：AI能理解元素关系，生成协调的画面，无需合成步骤
- `反向提示词`: 负面提示词（继承自顶层`反向提示词`）
- `风格`: 画面风格（继承自`项目元信息.全局风格`）
- `宽高比`: 默认"16:9"
- `分辨率`: 默认"1920x1080"
- `优先级`: 任务优先级

**叠加特效**（可选，10%的场景需要）：
- `叠加特效.启用`: 是否启用叠加特效
- `叠加特效.类型`: "静态图" | "关键帧"
- `叠加特效.提示词`: 特效提示词，如"光效、粒子特效、魔法光环"
- `叠加特效.反向提示词`: 特效负面提示词
- `叠加特效.风格`: 特效风格
- `叠加特效.优先级`: 特效优先级

**设计理念**：
- ✅ **AI优势**：AI擅长一次性生成完整、协调的内容，分层生成+合成反而增加复杂度和不协调风险
- ✅ **自动化优先**：减少合成步骤，减少对齐问题，提高自动化程度
- ✅ **灵活可选**：需要叠加特效时才启用，保持灵活性

#### 视频字段（视频生成任务配置）

**AI驱动优化方案**：充分利用AI的一次性生成能力，采用**完整视频+叠加特效+口型同步**的简化结构。

**基础配置**（默认，90%的场景）：
- `启用`: 是否启用视频生成
- `类型`: "动态视频" | "动效" | "口型同步"
- `提示词`: **完整动态提示词**，包含角色动作、背景动态、运镜等所有元素，AI一次性生成完整动态
  - 示例："2D漫风，动态视频，拉镜运镜，青云镇全景，顾白御剑凌空动作，剑尖朝前，色调偏暗，云层缓慢移动，天空动态"
  - **优势**：AI能理解元素关系，生成协调的动态，无需精确对齐
- `反向提示词`: 视频生成负面提示词（继承自顶层`反向提示词`）
- `运镜方式`: 运镜方式（继承自顶层`运镜方式`）
- `转场方式`: 转场方式（继承自顶层`转场方式`）
- `依赖`: 依赖的任务类型数组，如["图像", "音频", "TTS"]
- `帧率`: 默认24
- `优先级`: 任务优先级

**叠加特效**（可选）：
- `叠加特效.启用`: 是否启用叠加特效动画
- `叠加特效.类型`: "动态视频" | "动效"
- `叠加特效.提示词`: 特效动画提示词，如"光效动画、粒子特效动态"
- `叠加特效.反向提示词`: 特效负面提示词
- `叠加特效.优先级`: 特效优先级

**口型同步**（有对白时启用）：
- `口型同步.启用`: 是否启用口型同步
- `口型同步.类型`: "口型同步"
- `口型同步.依赖`: 依赖数组，如["音频.VO", "TTS"]
- `口型同步.优先级`: 口型同步优先级

**设计理念**：
- ✅ **AI优势**：AI擅长一次性生成完整、协调的动态，分层生成+合成反而增加对齐复杂度
- ✅ **自动化优先**：减少合成步骤，减少对齐问题，提高自动化程度
- ✅ **灵活可选**：需要叠加特效或口型同步时才启用，保持灵活性

### 依赖关系字段（新增）

| 子字段 | 类型 | 说明 |
|--------|------|------|
| `前置镜头` | array(number) | 依赖的镜号列表，用于任务调度 |
| `后置镜头` | array(number) | 被依赖的镜号列表 |
| `任务依赖` | object | 任务间的依赖关系，如视频依赖图像和音频 |

**用途**：建立任务依赖图（DAG），支持ComfyUI Workflow的智能调度和并行执行。

## 四、AI 流水线使用方式（增强版）

### 主控编排流程

1. **脚本解析与任务分解**
   - 读取主控脚本（v4.0格式）
   - 按镜头顺序解析，计算时间轴（如未提供则自动计算）
   - 为每个镜头生成详细任务清单，优先使用`任务配置`，若无则从顶层字段继承

2. **任务优先级排序**
   - P0必出 → 立即生成
   - P1核心 → 优先生成
   - P2过渡 → 可并行生成

3. **ComfyUI Workflow编排**
   - 根据`任务配置`创建对应的ComfyUI节点
   - 建立任务依赖图（DAG），支持并行执行独立任务
   - 按`依赖关系`调度任务执行顺序

4. **执行与监控**
   - 任务队列管理
   - 进度跟踪
   - 错误重试机制
   - 资源缓存（避免重复生成）

5. **合成与输出**
   - 按`时间轴`对齐所有资源
   - **音频混音**：合并五大流线（VO + BGM + 拟声 + 动态音效 + 氛围音效）
   - **图像合成**：
     - 基础：直接使用AI生成的完整画面（无需合成）
     - 叠加：如果有 `图像.叠加特效`，将特效叠加到完整画面上
   - **视频合成**：
     - 基础：直接使用AI生成的完整视频（无需合成）
     - 叠加：如果有 `视频.叠加特效`，将特效动画叠加到完整视频上
     - 口型：如果有 `视频.口型同步`，将口型动画叠加到角色动画上
   - 最终输出

### 各任务类型详细说明

#### 1. 图像生成
- **触发条件**：`生成任务` 含 **图像** 或 `图像.启用` 为 true
- **基础生成**（AI一次性生成完整画面）：
  - 提示词：`图像.提示词` 或 `AI提示词`（应包含角色、背景、道具等完整描述）
  - 反向提示词：`图像.反向提示词` 或 `反向提示词` 或 `项目元信息.默认反向提示词`
  - 风格：`图像.风格` 或 `项目元信息.全局风格`
  - 分辨率：`图像.分辨率`
  - 宽高比：`图像.宽高比`
  - **优势**：AI一次性生成完整、协调的画面，无需合成步骤
- **叠加特效**（可选）：
  - 如果 `图像.叠加特效.启用` 为 true，生成独立特效图层
  - 提示词：`图像.叠加特效.提示词`
  - 合成方式：叠加到完整画面上
- **可选增强**：用 `角色ID` 查角色设定，把立绘/服装等并入提示词

#### 2. 视频生成
- **触发条件**：`生成任务` 含 **视频** 或 `视频.启用` 为 true
- **基础生成**（AI一次性生成完整动态）：
  - 提示词：`视频.提示词` 或 `图像.提示词` 或 `AI提示词`（应包含角色动作、背景动态、运镜等完整描述）
  - 反向提示词：`视频.反向提示词` 或 `图像.反向提示词` 或 `反向提示词` 或 `项目元信息.默认反向提示词`
  - 运镜方式：`视频.运镜方式` 或 `运镜方式`
  - 转场方式：`视频.转场方式` 或 `转场方式`
  - 帧率：`视频.帧率`
  - **优势**：AI一次性生成完整、协调的动态，无需精确对齐
- **叠加特效**（可选）：
  - 如果 `视频.叠加特效.启用` 为 true，生成独立特效动画
  - 提示词：`视频.叠加特效.提示词`
  - 合成方式：叠加到完整视频上
- **口型同步**（有对白时）：
  - 如果 `视频.口型同步.启用` 为 true，生成口型动画
  - 依赖：`视频.口型同步.依赖`（如["音频.VO", "TTS"]）
  - 合成方式：叠加到角色动画上
- **依赖处理**：检查 `视频.依赖` 或 `视频.口型同步.依赖`，确保依赖任务已完成
- **连贯性**：可结合上一镜/下一镜做连贯性；口型可与 TTS 对齐

#### 3. TTS（VO流线）
- **触发条件**：`生成任务` 含 **TTS** 或 `音频.VO.启用` 为 true
- **输入参数**：
  - 台词：`音频.VO.台词` 或 `台词`
  - 角色ID：`音频.VO.角色ID` 或 `角色ID`
  - 情绪描述：`音频.VO.情绪描述` 或 `情绪描述`
  - 情绪向量：`音频.VO.情绪向量` 或 `情绪向量`
  - 音量：`音频.VO.音量`
- **输出**：WAV文件 + 时间戳（基于`时间轴.开始时间`）
- **上下文**：可把上一镜、下一镜的 `画面描述`+`台词` 作为情感上下文

#### 4. BGM流线
- **触发条件**：`生成任务` 含 **BGM** 或 `音频.BGM.启用` 为 true
- **输入参数**：
  - 类型：`音频.BGM.类型`（生成/库匹配/延续）
  - 提示词：`音频.BGM.提示词`（当类型为"生成"时使用）
  - 风格：`音频.BGM.风格`
  - 情绪描述：`音频.BGM.情绪描述`
  - 强度：`音频.BGM.强度`
  - 淡入/淡出时间：`音频.BGM.淡入时间` / `淡出时间`
- **输出**：BGM片段 + 淡入淡出时间点

#### 5. 拟声流线（Foley）
- **触发条件**：`生成任务` 含 **音效** 或 `音频.拟声.启用` 为 true
- **输入参数**：
  - 类型：`音频.拟声.类型`（生成/库匹配）
  - 提示词：`音频.拟声.提示词`（当类型为"生成"时使用）
  - 描述：`音频.拟声.描述` 或从 `音频设计` 解析
  - 触发时间：`音频.拟声.触发时间`（相对于镜头开始）
  - 音量：`音频.拟声.音量`
- **输出**：Foley音效文件 + 时间戳

#### 6. 动态音效流线
- **触发条件**：`音频.动态音效.启用` 为 true
- **输入参数**：
  - 类型：`音频.动态音效.类型`（转场/状态变化/信息暗示）
  - 提示词：`音频.动态音效.提示词`（用于AI生成转场或状态变化音效）
  - 描述：`音频.动态音效.描述`
  - 触发时间：`音频.动态音效.触发时间`
  - 音量：`音频.动态音效.音量`
- **输出**：动态音效文件 + 触发时间点

#### 7. 氛围音效流线
- **触发条件**：`音频.氛围音效.启用` 为 true
- **输入参数**：
  - 类型：`音频.氛围音效.类型`（生成/库匹配）
  - 提示词：`音频.氛围音效.提示词`（当类型为"生成"时使用）
  - 描述：`音频.氛围音效.描述` 或从 `音频设计` 解析
  - 强度：`音频.氛围音效.强度`
  - 循环：`音频.氛围音效.循环`
- **输出**：氛围音文件 + 循环参数

#### 8. 转场
- **触发条件**：`生成任务` 含 **转场** 或视频任务中包含转场
- **输入参数**：`转场方式`、前后镜画面描述
- **输出**：转场效果素材

### 编排与重跑策略

1. **任务调度**
   - 按 `镜号` 顺序执行；`优先级` 可用于排期（先 P0/P1 再 P2）
   - 根据 `依赖关系` 建立任务依赖图（DAG）
   - 支持并行执行独立任务

2. **资源复用**
   - 某镜只重跑图像时，仅对该镜执行 **图像** 任务，其它任务可复用已有资源
   - 使用资源缓存机制，避免重复生成相同内容

3. **错误处理**
   - 任务级重试机制
   - 失败任务记录，支持单独重跑
   - 依赖任务失败时，自动标记后续任务为待定状态

4. **时间轴同步**
   - 使用 `时间轴` 字段确保所有资源精确对齐
   - 音频混音时按时间轴合并五大流线
   - 视频合成时按时间轴对齐图像序列和音频

## 五、字段继承规则

为了简化填写，任务配置字段（`音频`、`图像`、`视频`）可以从顶层字段自动继承：

- **音频.VO**：优先使用 `音频.VO.*`，若无则继承顶层 `台词`、`角色ID`、`情绪描述`、`情绪向量`
- **音频.BGM**：优先使用 `音频.BGM.提示词`，若无则从 `音频设计` 解析或使用 `风格`+`情绪描述` 组合
- **音频.拟声/动态音效/氛围音效**：优先使用 `音频.*.提示词`，若无则使用 `音频.*.描述`，再则从 `音频设计` 解析
- **图像**：
  - 基础：优先使用 `图像.提示词`，若无则继承顶层 `AI提示词`
  - 叠加特效：优先使用 `图像.叠加特效.提示词`，若无则从 `图像.提示词` 提取特效部分
- **视频**：
  - 基础：优先使用 `视频.提示词`，若无则继承 `图像.提示词` 或顶层 `AI提示词`，并添加动态描述
  - 叠加特效：优先使用 `视频.叠加特效.提示词`，若无则从 `视频.提示词` 提取特效部分
  - `视频.反向提示词` 继承 `图像.反向提示词` 或顶层 `反向提示词`

**建议**：对于简单场景，只需填写顶层字段和`生成任务`即可；对于复杂场景，使用任务配置字段（`音频`、`图像`、`视频`）进行精细控制。

## 八、最佳实践

1. **时间轴计算**：如果未提供`时间轴`，工具应自动计算（从0开始累计各镜头时长）
2. **任务配置简化**：对于大多数镜头，只需填写顶层字段，`任务配置`可选；需要精细控制时再填写详细配置
3. **依赖关系**：视频任务通常依赖图像和音频，应在`任务配置.视频.依赖`中明确标注
4. **音频流线优先级**：VO > 核心音效（拟声、动态音效）> BGM > 氛围音效
5. **资源缓存**：相同参数的任务应复用缓存，避免重复生成
