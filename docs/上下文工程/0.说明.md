# 📘 项目文档放置指南（Cursor + AI 开发范式）

规范项目中各类文档的放置位置，使 AI 在生成代码、分析架构、执行任务时能正确加载上下文，并保持团队协作一致。

---

## 0. LLM 上下文加载顺序（核心概念）

Cursor 注入到模型的上下文顺序是：

1. **Rules（系统级）** — 宪法，不可违背  
2. **AGENTS.md（角色级）** — 部门规章，按需加载  
3. **README（项目级）** — 项目介绍，按需加载  
4. **当前对话内容** — 当前任务  
5. **相关代码片段** — Cursor 自动检索的事实依据  

| 层级 | 文档 | 作用 |
|------|------|------|
| 1 | Rules | 宪法（不可违背） |
| 2 | AGENTS.md | 角色规章（按需加载） |
| 3 | README | 项目介绍（按需加载） |
| 4 | Chat | 当前任务 |
| 5 | 代码 | 事实依据 |

---

## 1. Rules（系统级规则）

**位置：** `.cursor/rules/*.mdc`  
**作用：** 定义 AI 的"系统行为规则"，影响所有生成行为。本项目的规则文件（含 go-backend、flutter-frontend、fullstack-go-flutter、agent-logic 等）已统一放在 `.cursor/rules/`。

**加载时机：** 始终加载，每次都会注入

- 每一次 AI 生成代码时都会加载  
- 每一次 Chat 回复都会加载  
- 每一次编辑器内的自动补全 / 生成代码都会加载  
- 每一次 `@cursor edit`、`@cursor fix`、`@cursor explain` 都会加载  
- 每一次 Agent 执行任务时都会加载  

> Rules 是 Cursor 的「操作系统级规则」，永远在最前面。

**内容类型：**

- 代码风格规范  
- 架构原则  
- 命名约定  
- 禁止事项  
- 安全规则  
- 生成策略（如“优先使用 TypeScript”）

**不应包含：**

- 工具使用说明  
- 项目背景  
- API 文档  
- 角色说明

**一句话原则：**  
**Rules = 宪法，只放“行为规则”，不放项目内容。**

---

## 2. AGENTS.md（角色定义）

**位置：** `AGENTS.md` 或 `.cursor/agents/*.md`  
**作用：** 定义不同 AI 角色的职责、能力、可用工具。

**加载时机：** 按需加载

- 当你在 Chat 中选择某个 Agent（如 Architect、Coder、Reviewer）时  
- 当你在对话中使用 `@agent` 指令时  
- 当你在 CLI 中指定 agent 执行任务时  

> AGENTS.md 只有在你「调用某个角色」时才会加载。

**内容类型：**

- 架构师、代码生成器、测试工程师等角色的职责  
- 每个角色可使用的工具  
- 每个角色的工作方式与边界

**不应包含：**

- 项目说明  
- 工具详细文档  
- 系统级规则（应放 Rules）

**一句话原则：**  
**AGENTS = 角色说明书，只放“谁负责什么”。**

---

## 3. README（项目级上下文）

**位置：** 项目根目录 `README.md`  
**作用：** 提供项目的整体背景，让 AI 理解"这是一个什么项目"。

**加载时机：** 按需加载

- 当你在 Chat 中提到「根据项目」「根据 README」时  
- 当你让 AI 生成代码、创建文件、解释项目结构时  
- 当你在编辑器中生成代码时，Cursor 会自动把 README 作为项目背景注入  
- 当你使用 Ask Cursor 时  

> README 是「项目描述」，只有在任务需要理解项目时才会加载。

**内容类型：**

- 项目简介  
- 技术栈  
- 目录结构  
- 运行方式  
- 工具列表（仅概览）  
- 链接到 docs/ 的详细文档

**不应包含：**

- 详细工具说明  
- 角色定义  
- 行为规则

**一句话原则：**  
**README = 项目介绍，只放“项目是什么”。**

---

## 4. 工具使用说明文档（工具级知识）

**位置：** `docs/tools/*.md`  
**作用：** 记录工具的使用方式、参数、示例，供 AI 和开发者查阅。  
**内容类型：**

- 内部工具（build、deploy、脚本）  
- 外部工具（OpenClaw、ComfyUI、FFmpeg 等）  
- API 文档（如素材库、角色库、渲染服务）  
- 使用示例、参数说明

**不应放在：**

- Rules（会污染系统规则）  
- AGENTS（除非是某角色专属工具）  
- README（仅放概览即可）

**一句话原则：**  
**工具说明 = docs/tools，只放“如何使用工具”。**

## 6. 一页总结

| 文档类型   | 加载频率               | 放置位置        | 主要作用           | 不放什么       |
| ---------- | ---------------------- | --------------- | ------------------ | -------------- |
| **Rules**  | 每次对话/文件匹配时    | `.cursor/rules` | 规定「怎么写」     | 工具说明、项目内容 |
| **AGENTS** | 手动 @ 或任务规划时    | `AGENTS.md`     | 详述「做什么」     | 项目说明、系统规则 |
| **README** | 索引及全局检索时       | 根目录          | 告诉「是什么」     | 工具细节、规则、角色 |
| **工具说明** | 按需 @ 或检索时      | `docs/tools`    | 说明「如何用工具」 | 系统规则、角色定义 |

## 7. 上下文使用建议

- **精准触发 (globs)**：若希望某规则仅在特定路径生效，可在规则文件中设置 `globs`，如 `src/components/**/*` 或 `apps/web/**/*`。
- **启用 Composer Agent**：在 Cursor Settings 中开启 Composer Agent，AI 才能调用检索、终端等工具配合规则执行。
- **按需关联文档**：在对话中输入 `@rules/agent-logic`、`@docs/your-feature.md` 等，可强制将指定文档纳入上下文。本项目规则文件位于 `.cursor/rules/`，包含 `ai-development-guidelines.mdc`、`go-backend.mdc`、`flutter-frontend.mdc`、`fullstack-go-flutter.mdc`、`agent-logic.mdc`。

## 8. 规则应用范围（Apply to Specific Files）

在 Cursor 中编辑规则时，会看到 **「Apply to Specific Files」** 及右侧的 glob 模式输入框，用于控制该规则对哪些文件生效。

### 8.1 界面含义

- **Apply to Specific Files**：表示规则仅对匹配到的文件生效，而非全局。
- **Glob 模式**：文件路径匹配表达式，支持多个模式，用逗号分隔。

### 8.2 常见模式说明

| 模式 | 含义 |
|------|------|
| `**/*` | 项目根目录及所有子目录下的全部文件（默认全项目生效） |
| `**/*.dart` | 所有 Dart 文件 |
| `anime_ui/lib/**/*` | `anime_ui/lib` 下的所有文件 |
| `**/*_test.dart` | 所有测试文件 |
| `src/components/**/*` | `src/components` 下的所有文件 |

**符号含义：**

- `**`：匹配零个或多个目录层级
- `*`：匹配当前层级下任意字符

### 8.3 配置方式

- **在 .mdc 文件中**：在 frontmatter 中设置 `globs: **/*` 或 `globs: anime_ui/lib/**/*`。
- **在 Cursor 规则编辑界面**：在「Apply to Specific Files」右侧输入框中填写或修改模式，可添加多个。
