# 环境配置说明（Ubuntu 开发机）

本文档说明在新 Ubuntu 系统上为本项目（Go + Flutter 全栈）配置开发环境的步骤，以及一键脚本用法。

## 一、适用系统与前置条件

- **系统**：Ubuntu 22.04 LTS 或更高（含 Debian 11+）
- **权限**：部分步骤需要 `sudo`（安装系统依赖、Fcitx5+Rime）；Cursor、Go、Flutter 可仅安装到用户目录，无需 root
- **网络**：需能访问 go.dev、github.com、api2.cursor.sh 等

## 二、一键脚本（推荐）

项目提供脚本 `scripts/setup-ubuntu-dev.sh`，可自动完成：

| 组件 | 说明 | 安装位置 | 需 sudo |
|------|------|----------|--------|
| **用户目录** | 创建 `~/App`（软件）、`~/Dev`（代码） | `$HOME/App`、`$HOME/Dev` | 否 |
| **系统依赖** | curl, wget, git, unzip, xz-utils, zip, build-essential, libglu1-mesa, ca-certificates；可选 postgresql-client、redis-tools | 系统 | 是 |
| **SSH** | openssh-server，并设为开机自启 | 系统 | 是 |
| **远程桌面** | xrdp，并设为开机自启（可用 Windows 远程桌面连接） | 系统 | 是 |
| **Go** | 最新稳定版（从 go.dev/VERSION 获取） | `$INSTALL_DIR/go`，默认 `~/App/go` | 否 |
| **Flutter** | stable 分支（git clone） | `$INSTALL_DIR/flutter`，默认 `~/App/flutter` | 否 |
| **Cursor** | 官方 AppImage | `$INSTALL_DIR/bin/cursor.AppImage`，默认 `~/App/bin/` | 否 |
| **Fcitx5 + 中州韵(Rime)** | 输入法框架 + 朙月拼音·简化字（默认简体）+ Emoji | 系统包 + `~/.local/share/fcitx5/rime/` | 是 |

### 基本用法

```bash
# 克隆或进入项目目录后执行（会请求 sudo 密码）
bash scripts/setup-ubuntu-dev.sh
```

### 选项

```bash
# 指定安装根目录（默认为 ~/App）
bash scripts/setup-ubuntu-dev.sh --install-dir=$HOME/apps

# 跳过不需要的组件
bash scripts/setup-ubuntu-dev.sh --skip-cursor --skip-rime

# 不启用 SSH 与远程桌面自启动
bash scripts/setup-ubuntu-dev.sh --skip-ssh --skip-remote-desktop

# 仅安装系统依赖 + Go + Flutter（不装 Cursor、Rime、SSH、远程桌面）
bash scripts/setup-ubuntu-dev.sh --skip-cursor --skip-rime --skip-ssh --skip-remote-desktop

# 只预览将要执行的步骤
bash scripts/setup-ubuntu-dev.sh --dry-run
```

| 选项 | 说明 |
|------|------|
| `--install-dir=DIR` | 安装根目录，Go/Flutter/Cursor 均在其下，默认 `~/App` |
| `--skip-cursor` | 不安装 Cursor |
| `--skip-go` | 不安装 Go |
| `--skip-flutter` | 不安装 Flutter |
| `--skip-rime` | 不安装 Fcitx5+Rime |
| `--skip-ssh` | 不安装/不启用 SSH 服务自启动 |
| `--skip-remote-desktop` | 不安装/不启用 xrdp 远程桌面自启动 |
| `--skip-sysdeps` | 不安装系统依赖包 |
| `--dry-run` | 仅打印步骤，不实际执行 |

### 脚本执行后必做

1. **使 PATH 生效**
   ```bash
   source ~/.profile
   ```
   或重新打开终端 / 重新登录。

2. **验证 Go 与 Flutter**
   ```bash
   go version
   flutter doctor
   ```
   首次运行 `flutter` 会下载 Dart SDK，需保持网络畅通。

3. **Fcitx5/Rime 生效**  
   需**注销并重新登录**（或重启），环境变量与输入法框架才会生效。  
   登录后若托盘无输入法图标，可执行：`fcitx5 -d`  
   在 `fcitx5-configtool` 中添加「中州韵(Rime)」，默认 Ctrl+Space 切换中英文。

4. **Cursor 启动方式**  
   若安装到 `~/App/bin/cursor.AppImage`（默认），在 PATH 生效后可直接执行：
   ```bash
   cursor.AppImage
   ```

5. **SSH 与远程桌面**  
   脚本会安装并**开机自启** SSH（openssh-server）与 xrdp。  
   - SSH：`ssh 用户@本机IP`  
   - 远程桌面：在 Windows 上打开「远程桌面连接」，输入本机 IP，用当前 Ubuntu 用户名与密码登录。

## 三、PATH 与安装位置汇总

脚本会创建并使用的目录：

- **`~/App`**（默认安装根目录）：Go、Flutter、Cursor 均在其下。
- **`~/Dev`**：建议用于存放代码仓库（脚本会自动创建）。

脚本会向 `~/.profile` 追加（若尚未存在）：

- `export PATH="$PATH:<INSTALL_DIR>/go/bin"`
- `export PATH="$PATH:<INSTALL_DIR>/flutter/bin"`
- `export PATH="$PATH:<INSTALL_DIR>/bin"`（Cursor AppImage 所在目录）

默认 `INSTALL_DIR=~/App` 时：

- Go：`~/App/go`
- Flutter：`~/App/flutter`
- Cursor：`~/App/bin/cursor.AppImage`
- 代码目录：`~/Dev`（仅创建目录，不写入 PATH）

## 四、Go / Flutter 开发建议依赖（脚本已包含或可选）

- **脚本已安装**：curl, wget, git, unzip, xz-utils, zip, build-essential, libglu1-mesa, ca-certificates；可选 postgresql-client、redis-tools（本项目后端使用 PostgreSQL 与 Redis）。
- **Android 开发**：若需运行 Flutter Android 应用，需另行安装 Android Studio 或 Android SDK/NDK，并运行 `flutter doctor --android-licenses`。
- **Chrome/Web**：Flutter Web 需要 Chrome/Chromium，`flutter run -d chrome` 前请确保已安装。
- **本项目后端**：PostgreSQL、Redis 需在本地或远程可用；配置见 `anime_ai/config.yaml.example` 与 README。

### 4.1 后端本地服务（Redis / PostgreSQL）

若在本机跑 Go 后端且需**异步任务队列（Asynq）**与**数据库**，需安装并启动 Redis 与 PostgreSQL 服务端（一键脚本只安装了客户端工具 `redis-tools`、`postgresql-client` 便于调试）。

**安装（需在终端执行，会提示输入 sudo 密码）：**

```bash
sudo apt-get update -y
sudo apt-get install -y redis-server postgresql
```

**创建数据库用户与库（可选，默认示例用户 yikai / 密码 mayikai）：**

```bash
sudo -u postgres psql -c "CREATE USER yikai WITH PASSWORD 'mayikai' CREATEDB;"
sudo -u postgres createdb -O yikai ai_anime
# 应用 schema（在项目 anime_ai 目录下）
PGPASSWORD=mayikai psql -h localhost -U yikai -d ai_anime -f sch/schema.sql
```

或直接执行项目脚本：`cd anime_ai && ./scripts/init_db.sh`

**启动服务：**

```bash
# 启动 PostgreSQL（默认监听 5432）
sudo service postgresql start

# 启动 Redis（默认监听 6379），后台运行
redis-server --daemonize yes
```

**可选：开机自启**

```bash
sudo systemctl enable redis-server
sudo systemctl enable postgresql
```

之后在项目根目录执行 `cd anime_ai && go run .` 时，Redis 与 PostgreSQL 可用则 Asynq Worker 会正常启动；若未安装或未启动，后端会优雅降级（仅打印 WARN，HTTP 服务照常监听）。

## 五、故障排除

- **Go/Flutter 命令找不到**  
  执行 `source ~/.profile` 或重新登录，确认 `echo $PATH` 中包含 `~/App/go/bin` 与 `~/App/flutter/bin`（或你设置的 `INSTALL_DIR`）。

- **Flutter 提示缺少 curl**  
  脚本会安装 curl；若未运行脚本的系统依赖步骤，请手动：`sudo apt-get install -y curl`。

- **Fcitx5 登录后不生效**  
  确认已注销/重登；Wayland 下检查 `~/.config/environment.d/fcitx5.conf` 是否存在且被桌面环境加载。

- **Cursor AppImage 无法运行**  
  部分环境需加 `--no-sandbox`：`./cursor.AppImage --no-sandbox`（仅当正常启动失败时使用）。

- **Rime 部署/配置**  
  若使用项目内配置，脚本会从 `config/rime/default.custom.yaml` 复制到 `~/.local/share/fcitx5/rime/`；否则使用脚本内嵌的默认方案（朙月拼音·简化字 + Emoji，**默认简体**）。修改配置后可在托盘图标中「重新部署」。
